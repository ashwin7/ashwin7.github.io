Work Process

As a human being, I'm work in process.

As with any software project, I begin with an idea and a purpose. Google Keep and Trello is utilized to connect the dots. When an idea is refined enough, I do adequate visual research where I make myself familiar with product's industry and target users, I make sketches using old fashioned pen & paper or Balasmiq Mockups. After multiple iterations, a Mockup is designed using Photoshop, Illustrator and After Effects (for demonstrating UI animations).

1. Idea
2. Sketch
3. Design
4. Develop
5. Test
6. Launch

After the design is finalized, I code the front-end and back-end in parallel using appropriate technologies. The focus is always on what problems we need to solve instead of which technology to use. If a technology feels limiting, it can be replaced with another. I've hands on experience with MySQL, Java and Python. Code is tested thoroughly and deployed. I prefer 'Apache Tomcat' but 'nginx' also works.
